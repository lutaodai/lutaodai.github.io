Sequential Computation along Sampling Points to Bypass OOM Error of Integrated Gradients

Integrated Gradients method [1] requires sampling points between the input and a baseline . As a result, the input to the module is expanded times, and so is the backward gradient calculation, which often leads to a Out-Of-Memory (OOM) errors. This post modifies the Captum's implementation on Integrated Gradients to address this issue. The main idea is to divide the -step approximation into mini-batches and evaluate them sequentially.

The modified function can be naturally extended to other gradient-based attribution methods (those with an n_steps argument).

Codes can be found on Github.


Usage

The modifications are based on Captum's implementation and can be found on Github.

Note that - pred_func takes x or x_prime (mandatory) and additional_forward_args (optional) as inputs. If there are embedding layers, x and x_prime should be the embedded features instead of indices. - n_steps is not necessarily required to be divisible by n_chunk. - when n_chunk is 1, the implementation is identical to the Captum implmentation
